In JavaScript, prototype and __proto__ serve different but related purposes in the inheritance model. Here’s a breakdown:

prototype

  • prototype is a property of constructor functions (such as function MyFunc() {} or ES6 class declarations).
  • This prototype property is an object that gets attached as the __proto__ (or [[Prototype]]) of instances created by that constructor.
  • Any properties or methods added to MyFunc.prototype will be accessible to all instances of MyFunc.

Example:

function MyFunc() {} MyFunc.prototype.sayHello = function() { console.log("Hello"); }; const instance = new MyFunc(); instance.sayHello(); // "Hello"

__proto__

  • __proto__ is a property of an object instance, pointing to the object’s prototype (its "parent" object in the prototype chain).
  • This __proto__ link allows instances to inherit properties and methods from their constructor’s prototype.
  • __proto__ itself is not standardized as a property but is commonly supported; it essentially gives access to the object's internal [[Prototype]].

Example:

function MyFunc() {} const instance = new MyFunc(); console.log(instance.__proto__ === MyFunc.prototype); // true

Key Differences

Ownership: prototype is a property of a constructor function. __proto__ is a property of an instance. Purpose: prototype defines properties and methods to be shared by instances created by the constructor. __proto__ connects an instance to its prototype for property lookup. Usage: You use prototype when defining shared properties/methods for instances. __proto__ is generally used for inspection and is rarely modified directly.

function Animal() {} Animal.prototype.speak = function() { console.log("Animal sound"); }; const dog = new Animal(); dog.speak(); // "Animal sound" // dog.__proto__ points to Animal.prototype console.log(dog.__proto__ === Animal.prototype); // true

In summary, prototype is used to set up inheritance, while __proto__ is a link to an object’s prototype, enabling inheritance to work.
