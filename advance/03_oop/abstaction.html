Abstaction

Abstraction is the process of simplifying complex systems by focusing only on the relevant details and hiding the irrelevant ones. It involves creating a simplified model of something more complex.

Purpose:
To expose only the essential features of an object, minimizing the amount of detail needed to understand it.

How Itâ€™s Achieved:
Through abstract classes and interfaces. These define a structure without detailing the implementation.

Example:
A Car class could have abstract methods like start() and stop(), representing actions that any car can perform without specifying how they work.

// Example in JavaScript using an abstract-like structure class Car { start() { throw "Abstract method! Implement in subclass"; } stop() { throw "Abstract method! Implement in subclass"; } } class ElectricCar extends Car { start() { console.log("Electric car starting..."); } stop() { console.log("Electric car stopping..."); } } const myCar = new ElectricCar(); myCar.start(); // "Electric car starting..."

Here, the Car class defines the actions (start, stop) any car should have, but the exact workings are left to the subclasses.
